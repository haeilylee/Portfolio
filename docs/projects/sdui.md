## 배경

Shadcn은 web 기반 베이스이기 때문에 컴포넌트 체계를 모두가 이해할 수 있는 구조로 세우고, 실제로 대응할 수 있는 형태로 정리해야 했어요.

## 문제

서버 드리븐 UI 전환에 맞춰 디자인 시스템 컴포넌트를 재정의하고, 개발/디자인 팀 모두가 참조할 수 있는 문서 체계를 구축했어요.
